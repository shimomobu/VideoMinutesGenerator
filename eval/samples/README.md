# eval/samples — 評価サンプル一覧

評価ケースのメタ情報を管理する。動画/音声ファイル本体はリポジトリに含めない（evaluation-plan.md §7.2）。

## ケース一覧

| CASE ID | 種別 | 音声種類 | 時間目安 | 状態 |
|---------|------|---------|---------|------|
| CASE-01 | 標準ケース（技術定例） | 無音ダミー（transcript 手動上書き） | 10〜15分 | meta 定義済み・run-6 PASS |
| CASE-02 | 大規模ケース（意思決定会議） | 実音声（業務側提供待ち） | 25〜30分 | 未定義 |
| CASE-03 | 短尺ケース（短い打ち合わせ） | 合成音声（open-jtalk） | 1〜2分 | meta 定義済み・未実行 |
| CASE-04 | 低品質ケース（雑音・エコー） | 実音声（業務側提供待ち） | 10〜20分 | 未定義 |
| CASE-05 | 境界ケース（無音・ノイズ） | ffmpeg 生成（開発側で作成可） | 〜2分 | meta 定義済み・未実行 |

## 音声ファイルの保管方針

- 動画/音声ファイルは各自のローカル環境に保管し、リポジトリには含めない
- `.gitignore` に `eval/samples/**/*.mp4` 等が設定済み
- ファイルパスは `meta.yaml` には記載せず、実行時に `--input` で指定する
